Project overview

Socket_cpp is a lightweight, modern C++ library that simplifies TCP socket programming for client-server applications. Built with C++17 features, it provides a clean object-oriented interface for network communication while handling low-level socket operations and connection management.

Core Features

  • Simple TCP client-server socket communication
  • Modern C++17 design with smart pointer management
  • Cross-platform POSIX socket support (Linux/Unix)
  • Thread-safe operations with multi-threading support
  • Comprehensive CMake build system with separate client/server libraries
  • Robust error handling and connection management

Architecture

The library is structured as separate client and server static libraries, each with their own CMake configuration. The unified header file socket.hpp provides a consistent API, while platform-specific implementations handle the underlying socket operations.

Quick Usage Example

Basic client-server implementation demonstrating the API:

Server Implementation

#include "socket.hpp"

int main() {
    // Create server socket on localhost:12345
    Socket::Ptr server = std::make_shared<Socket>("127.0.0.1", 12345);
    server->socket_connect();
    
    // Send message to client
    std::string msg = "Hello from server!";
    server->socket_send(msg);
    
    // Receive response from client
    std::string response = server->socket_receive();
    std::cout << "Client response: " << response << std::endl;
    
    server->socket_close();
    return 0;
}

Client Implementation

#include "socket.hpp"

int main() {
    // Connect to server
    Socket::Ptr client = std::make_shared<Socket>("127.0.0.1", 12345);
    client->socket_connect();
    
    // Receive message from server
    std::string message = client->socket_receive();
    std::cout << "Server says: " << message << std::endl;
    
    // Send response back
    std::string reply = "Hello from client!";
    client->socket_send(reply);
    
    client->socket_close();
    return 0;
}

Building & Installation

Complete build instructions using CMake:

# 1. Clone the repository
git clone https://github.com/aarwitz/Socket_cpp.git
cd Socket_cpp

# 2. Build client library
cd Client
mkdir build && cd build
cmake ..
make
sudo make install  # Installs libclient.a and socket.hpp

# 3. Build server library  
cd ../../Server
mkdir build && cd build
cmake ..
make
sudo make install  # Installs libserver.a

# 4. Build example applications
cd ../SocketEx/clientSrc
mkdir build && cd build
cmake ..
make  # Creates clientEx executable

cd ../../serverSrc
mkdir build && cd build
cmake ..
make  # Creates serverEx executable

The build system creates separate static libraries for client and server functionality, allowing applications to link only the required components.

Technical Implementation

  • Smart Pointer Management: Uses std::shared_ptr for automatic memory management
  • RAII Design: Socket resources automatically cleaned up on object destruction
  • Thread Safety: Designed for multi-threaded applications with proper synchronization
  • Error Handling: Comprehensive connection state management and error reporting
  • Platform Abstraction: POSIX socket API with Linux/Unix compatibility
  • Modular Build: Separate client/server libraries for optimized linking
